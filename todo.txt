Hệ thống sẽ cho nhiều người dùng sài
Sẽ có một dictionary tổng, với key là index và value là cat object
Bất cứ khi nào user yêu cầu một bài kiểm tra thì sẽ đưa một cat object mới vào trong dictionary, gửi lại index vô trong local storage của user để lưu. Sau khi lưu thì dùng index đó để request làm bài của cat object có index tương ứng.

Với việc làm bài:
- Hiện câu hỏi lên cho user, tạm ngưng, để đó đợi user làm bước tiếp theo.
- Khi user ấn vào nút trả lời câu hỏi thì backend sẽ nhận và đưa câu trả lời vào cho cat object
- Sau đó, cat object sẽ ước lượng lại các tham số của user đó, và quyết định xem có nên đi tiếp hay không.
- Nếu tiếp thì trả về thông tin gì đó để front end request câu hỏi kế, còn không thì cũng báo để front end kết thúc bài kiểm tra.
